Installation¶

Installation from Source Code¶

Requirements¶

Since Apache AGE is an extension of PostgreSQL, its build system is based on Extension Building Infrastructure that PostgreSQL provides. Therefore, pg_config (typically the first one in your PATH environment variable) for the PostgreSQL installation is used to build it.

Apache AGE 0.2.0 supports PostgreSQL 11.

Getting The Source¶

The Apache AGE 0.2.0 sources can be obtained from its GitHub repository: bitnine-oss/AgensGraph-Extension/releases.

Installation Procedure¶

To build and install Apache AGE, run the following command in the source code directory of Apache AGE.

$ make install

Since Apache AGE will be installed in the directory of the PostgreSQL installation, proper permissions on the directory are required.

Run the following statements in psql to create and load Apache AGE in PostgreSQL.

=# CREATE EXTENSION age; -- run this statement only once
CREATE EXTENSION
=# LOAD 'age';
LOAD
=# SET search_path = ag_catalog, "$user", public;
SET

When Apache AGE is being loaded, it installs post_parse_analyze_hook, set_rel_pathlist_hook, and object_access_hook to analyze and execute Cypher queries.
